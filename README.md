## 先说说 StarUML 7 到底多能打

很多人印象里 StarUML 就是个"画 UML 图的"，但 v7 版本早就不只是画图了。

它能支持的图表语言包括：UML 2.x 全系（类图、序列图、用例图、状态图、活动图等）、ER 图、流程图、数据流图、思维导图、C4 模型，专业版还额外支持 SysML、BPMN、UI 线框、AWS 架构图、GCP 架构图、Azure 架构图。

一个项目里可以同时用多种建模语言，不用开七八个工具。

v7 的几个新特性值得单独说一下：

- **AI 接入 via MCP Server**：支持用自然语言指令生成和调整模型图，跑通 AI 辅助建模的完整流程
- **Mermaid 语法直接转图**：写几行文字就能生成时序图、流程图，快速原型神器
- **设备绑定授权机制**：比旧版更严格，也更稳定，正版用户体验更顺畅
- **支持 Azure 架构图**（v7 新增）：做云架构的朋友应该会用得上

跨平台方面，Windows、macOS（含 Apple Silicon）、Linux 三端都支持，一套授权绑定设备，不同系统之间切换没有障碍。

## 用起来怎么样

作为一个跑在 Electron 框架上的工具，StarUML 的性能表现在同类产品里算中等偏上，启动和响应速度说不上飞快，但日常使用不会有明显卡顿。

界面支持亮色和暗色两套主题，这点很重要——长时间盯着白色背景做架构图，眼睛会扛不住。快捷建模这块做得也不错，Quick Edit 模式下可以一次性批量创建子类、接口这些关联关系，不用一个个拖来拖去。

模型数据用 JSON 格式存储，这意味着可以直接拿 CLI 工具配合模板自动生成代码、文档、图片，搭 CI/CD 流水线也没问题。

需要提一下的局限是：**软件界面目前不支持中文**。如果你的团队全员中文，这一点可能有点磕脚，但主要操作区域没有太多文字，适应周期不长。

扩展生态这块，StarUML 有独立的插件市场，能找到 Java、Python、C#、C++ 的代码生成扩展，不少插件是开源的，可以 fork 自己改。
